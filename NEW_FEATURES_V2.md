### 🔐 СИСТЕМА АУТЕНТИФИКАЦИИ И АВТОРИЗАЦИИ

#### Backend

#### Frontend
- ✅ Страница Register (регистрация)
- ✅ Страница Login (вход)
- ✅ Auth Service для управления токенами
- ✅ ProtectedRoute компонент для защиты страниц
- ✅ Сохранение токена в localStorage
- ✅ Автоматическое добавление токена в запросы

### 🚫 ОГРАНИЧЕНИЯ ДОСТУПА

- ✅ Dashboard (мои ссылки) - только для авторизованных пользователей
- ✅ Statistics (статистика) - только для авторизованных пользователей
- ✅ Создание ссылок - только для авторизованных пользователей
- ✅ Гость может видеть только главную страницу (Home)

### 👨‍💼 АДМИНИСТРАТОРСКАЯ ПАНЕЛЬ (новое)

#### Функции админа
1. **Управление пользователями:**
   - Просмотр списка всех пользователей
   - Просмотр деталей пользователя
   - Удаление пользователей (с автоматическим удалением их ссылок)
   - Изменение роли пользователя (admin/user)

2. **Управление ссылками:**
   - Просмотр всех ссылок в системе
   - Просмотр ссылок конкретного пользователя
   - Изменение статуса ссылких (active/blocked/inactive)
   - Удаление любых ссылок
   - Просмотр информации о создателе ссылки (user_id)

#### Доступ к админ-панели
- URL: `/admin`
- Ограничено: только для администраторов (role = 'admin')
- Автоматический редирект: если обычный пользователь попытается перейти на `/admin`, он будет перенаправлен на `/dashboard`

### 📊 API ENDPOINTS (новые)

#### Аутентификация
```
POST /api/auth/register          - Регистрация
POST /api/auth/login             - Вход
GET  /api/auth/verify            - Проверка токена (требует токен)
POST /api/auth/logout            - Выход
```

#### Пользовательские ссылки
```
GET  /api/links/user/my-links    - Мои ссылки (требует авторизацию)
POST /api/links                  - Создать ссылку (требует авторизацию)
```

#### Администраторские функции
```
GET    /api/admin/users                 - Список всех пользователей
GET    /api/admin/users/:userId         - Деталь пользователя
DELETE /api/admin/users/:userId         - Удалить пользователя
PATCH  /api/admin/users/:userId/role    - Изменить роль пользователя
GET    /api/admin/links                 - Все ссылки (с info о user_id)
DELETE /api/admin/links/:id              - Удалить ссылку
PATCH  /api/admin/links/:id             - Изменить статус ссылки
```

### 🔄 ИЗМЕНЕНИЯ В СУЩЕСТВУЮЩЕЙ ФУНКЦИОНАЛЬНОСТИ

1. **Создание ссылок (POST /api/links)**
   - До: Мог создать любой
   - После: Требует аутентификации, userId берется из JWT токена

2. **Dashboard**
   - До: Показывал все ссылки в системе
   - После: Показывает только ссылки текущего пользователя

3. **Home страница**
   - До: Все могли создавать ссылки
   - После: Гость видит сообщение об ошибке и кнопку для входа при попытке создать ссылку

4. **Навигация**
   - До: `Home | Dashboard | Statistics`
   - После: 
     - Для гостей: `Home | Login | Register`
     - Для пользователей: `Home | Dashboard | Statistics | [Username] | Logout`
     - Для админов: `Home | Admin Panel | [Username] | Logout`

### 📝 ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

#### 1. Регистрация
```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com", "username":"john", "password":"password123"}'
```

#### 2. Вход
```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com", "password":"password123"}'
```

#### 3. Создание ссылки (отправить токен)
```bash
curl -X POST http://localhost:3000/api/links \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer [your_token]" \
  -d '{"originalUrl":"https://example.com/very/long/url"}'
```

#### 4. Получить ссылки пользователя
```bash
curl -X GET http://localhost:3000/api/links/user/my-links \
  -H "Authorization: Bearer [your_token]"
```

#### 5. Удалить пользователя (админ)
```bash
curl -X DELETE http://localhost:3000/api/admin/users/2 \
  -H "Authorization: Bearer [admin_token]"
```

### ⅰ️ ВАЖНЫЕ ЗАМЕЧАНИЯ

1. **Защита консольного админ-интерфейса**
   - Старая консольная админ-панель (`npm run admin`) всё ещё существует
   - Новая веб-панель рекомендуется использовать вместо неё
   - Обе работают с одной БД, но веб-панель имеет лучший интерфейс

2. **JWT токены**
   - Каждый токен действует 7 дней
   - Токен хранится в localStorage браузера
   - При перезагрузке страницы автоматически использует сохранённый токен

3. **Безопасность паролей**
   - Пароли хешируются с bcryptjs (cost = 10)
   - Минимум 6 символов
   - Не передаются в открытом виде

4. **Гостевой доступ**
   - Гость может просмотреть главную страницу (Home)
   - Гость может переходить по коротким ссылкам (редирект работает для всех)
   - Гость НЕ может создавать ссылки, видеть Dashboard или Statistics
