FEATURE 2: PREVENT RE-SHORTENING OF ALREADY-SHORTENED URLS
═══════════════════════════════════════════════════════════════════════════════

File Modified: backend/src/controllers/LinkController.js

Changes:
  + Added: Static method isShortenedUrl() to detect shortened URLs
  + Modified: createLink() method now validates input URL
  + Logic: 
    - Extracts domain and path from URL
    - Uses regex /^[a-zA-Z0-9_-]{7}$/ to detect 7-char short codes
    - Only blocks URLs with our server domain + 7-char path
    - Allows same URL from different domains (not our service)

Test Results: ✅ ALL PASS
  
  Test 2.1: Blocked shortened URL
    Input: {"originalUrl":"http://localhost:3000/jX8UmgL"}
    Expected: 400 error
    Actual: {"error":"Cannot shorten an already shortened URL"}
    Status: ✅ PASS
  
  Test 2.2: Allowed normal URL
    Input: {"originalUrl":"https://www.wikipedia.org/wiki/Shortened_URL"}
    Expected: 201 created
    Actual: Created with shortCode "zBIkX71"
    Status: ✅ PASS
  
  Test 2.3: Different domain shortened URL
    Input: {"originalUrl":"https://short.link/ABC1234"}
    Expected: 201 created (not our domain)
    Status: ✅ PASS
  
  Test 2.4: Invalid pattern (8 characters)
    Input: {"originalUrl":"http://localhost:3000/INVALID1"}
    Expected: 201 created (not 7 chars)
    Status: ✅ PASS

Live Test:
  curl -X POST http://localhost:3000/api/links \
    -H "Content-Type: application/json" \
    -d '{"originalUrl":"http://localhost:3000/jX8UmgL"}'
  
  Expected Response:
  {"error":"Cannot shorten an already shortened URL"}
