# Веб-сервис для создания коротких ссылок и генерации QR-кодов

## Введение

Данная работа описывает разработку полнофункционального веб-сервиса для сокращения URL-адресов и генерации QR-кодов. Проект аналогичен популярным сервисам сокращения ссылок, таким как Bitly, и предоставляет интуитивное веб-приложение для работы с ссылками, а также администраторскую консоль для управления системой.

Проект состоит из трёх основных компонентов: серверной части на Node.js с Express, веб-интерфейса на React, и административной консоли для терминала. Система отслеживает посещения по укороченным ссылкам, различает переходы по QR-кодам и прямым ссылкам, и сохраняет подробную статистику использования.

### Инструкция по использованию системы

#### Часть 1: Инструкция для обычного пользователя

Веб-сервис предназначен для быстрого сокращения длинных URL-адресов и создания QR-кодов. Вот пошаговое руководство для работы с сайтом.

**Запуск и вход на сайт**

1. Откройте веб-браузер (Chrome, Firefox, Safari или другой)
2. Перейдите на адрес: http://localhost:5173
3. Вы увидите стартовую страницу с полем для ввода ссылки

**Основное окно для сокращения ссылок (Home)**

На главной странице находится большое поле ввода с надписью "Введите вашу длинную ссылку сюда". Это основной инструмент для работы с ссылками.

**Как создать короткую ссылку**

1. Найдите поле ввода в центре страницы
2. Скопируйте длинную ссылку из браузера или где-либо ещё
3. Вставьте её в поле ввода (Ctrl+V или Cmd+V)
4. Нажмите кнопку "Сгенерировать короткую ссылку"
5. Система за несколько миллисекунд создаст сокращённый код
6. Вы увидите блок с результатом, содержащий:
   - Оригинальную ссылку (при клике на стрелку она разворачивается для просмотра)
   - Короткую ссылку (например: http://localhost:3000/abc123)
   - QR-код и кнопку для скачивания

**Работа с обработанной ссылкой**

После создания короткой ссылки вы увидите:

1. Поле с исходной ссылкой - кликните на стрелку рядом, чтобы раскрыть и увидеть полный адрес
2. Кнопка "Копировать в буфер обмена" рядом с короткой ссылкой - один клик и ссылка скопирована
3. QR-код в виде чёрно-белого квадрата - его можно отсканировать телефоном
4. Кнопка "Скачать QR-код" для сохранения QR-кода на компьютер

**Навигация по сайту**

В верхней части сайта находится меню с тремя разделами:
- Home - главная страница для создания ссылок
- Dashboard - список всех ваших созданных ссылок
- Statistics - подробная статистика по ссылкам

**Просмотр всех ссылок (Dashboard)**

1. Кликните на пункт "Dashboard" в меню
2. Вы увидите таблицу со всеми созданными ссылками
3. В таблице отображаются:
   - Короткий код (например: abc123)
   - Исходная ссылка
   - Количество переходов
   - Статус ссылки (активна/заблокирована)

**Просмотр статистики (Statistics)**

1. Перейдите на вкладку "Statistics"
2. Введите номер ссылки или её короткий код в поле поиска
3. Нажмите кнопку поиска
4. Вы увидите подробное отчёт:
   - Количество всех переходов
   - Переходов по QR-коду
   - Переходов по прямой ссылке
   - QR-код для повторного использования
   - График переходов по дням
   - Исходная ссылка (нажмите на стрелку чтобы раскрыть)

**Использование QR-кода**

1. После создания ссылки вы получите QR-код
2. Его можно поделиться в социальных сетях, мессенджерах
3. Люди, отсканировав QR-код на телефоне, попадут на вашу исходную ссылку
4. Каждый такой переход будет учтён как "переход по QR-коду" в статистике

**Диагностика проблем**

Если страница не загружается:
- Убедитесь, что вы вводите правильный адрес: http://localhost:5173
- Проверьте, что сервер запущен (смотрите раздел "Установка и запуск" ниже)
- Попробуйте обновить страницу (F5 или Cmd+R)
- Закройте и откройте браузер заново

## Идея и актуальность проекта

В современном интернете URL-адреса стали всё более длинными и сложными. Длинные ссылки с множеством параметров, символов и цифр труднодоступны для запоминания и обмена. Если вы ходите отправить ссылку в мессенджер, социальную сеть или просто рассказать её словесно, длинный URL становится проблемой.

Проект "Сервис сокращения ссылок с QR-кодами" решает эту проблему. Он позволяет превратить длинную и запутанную ссылку в короткий код из 7 символов, который легко запомнить, скопировать и поделиться.

Дополнительно, система генерирует QR-коды. Это специальные изображения, которые можно отсканировать камерой телефона. При сканировании QR-кода телефон автоматически открывает нужную ссылку. Это особенно удобно для маркетинга, реклам и печатных материалов.

Система также отслеживает статистику: сколько раз нажали на короткую ссылку, сколько раз отсканировали QR-код, когда это произошло. Эта информация полезна для понимания того, насколько популярна ваша ссылка.

## Анализ предметной области

Предметная область включает несколько ключевых концепций:

Сокращение ссылок это преобразование длинного URL в короткий код, который при переходе редиректит на исходный адрес. Ключевой вызов здесь это гарантировать уникальность кода и избежать коллизий, когда два разных URL получают одинаковый код.

QR-коды это двумерные штрих-коды, которые содержат информацию. В нашем случае QR-код содержит короткую ссылку. Сканирование QR-код на телефоне автоматически открывает браузер с этой ссылкой.

Аналитика и статистика позволяют отслеживать, как часто используется ссылка, откуда приходят переходы (QR или прямая ссылка). Эти данные полезны для маркетинга и анализа.

Безопасность и защита от спама это критичный аспект. Система должна защищаться от попыток создать слишком много ссылок за короткое время, фильтровать вредоносные ссылки и предотвращать злоупотребления.

Администрирование и управление описывает способность администратора просматривать все ссылки, блокировать подозрительные, удалять старые, и просматривать статистику использования системы.

## Описание архитектуры системы

Система состоит из четырёх основных компонентов, взаимодействующих между собой.

На самом верхнем уровне находятся клиентские приложения: веб-сайт, который пользователи открывают в браузере, и консольная программа для администраторов. Оба эти интерфейса отправляют запросы на серверную часть.

В центре системы находится сервер на Node.js с Express. Это программа, которая слушает входящие запросы, обрабатывает их, обращается к базе данных, и отправляет ответы обратно клиентам. Сервер организован в несколько слоёв:

- Слой маршрутов определяет какой запрос куда должен идти
- Слой контроллеров содержит логику обработки каждого типа запроса
- Слой сервисов содержит основную бизнес-логику: генерация кодов, создание QR-кодов, проверка на спам
- Слой моделей описывает работу с базой данных

Под сервером находится база данных PostgreSQL. Это программа которая хранит все данные: информацию о ссылках, статистику переходов, информацию о пользователях. База данных организована в несколько таблиц.

Таблица "ссылки" хранит основную информацию о каждой сокращённой ссылке: её уникальный ID, короткий код, исходный URL, статус (активна или заблокирована).

Таблица "переходы" хранит информацию о каждом переходе: какая ссылка была использована, когда это произошло, с какого IP-адреса, какой браузер использовался, был ли это переход по QR-коду или по прямой ссылке.

Таблица "пользователи" хранит информацию о людях которые создают ссылки, если система требует регистрации.

Таблица "ограничения скорости" используется для защиты от спама и отслеживает сколько запросов пришло с каждого IP-адреса.

Веб-интерфейс написан на React и состоит из нескольких страниц. Главная страница имеет поле ввода для создания нового URL и отображает результат. Dashboard показывает список всех созданных ссылок. Statistics позволяет поиск ссылки и просмотр подробной статистики.

## Описание интерфейсов

Веб-интерфейс состоит из трёх основных страниц и навигационного меню.

На главной странице в центре располагается крупное поле ввода "Введите свой длинный URL". После ввода ссылки и нажатия кнопки "Сгенерировать" система выводит результат в блоке ниже. Этот блок содержит исходный URL с кнопкой разворачивания, сгенерированный короткий код с кнопкой копирования, и QR-код с кнопкой загрузки.

На странице Dashboard в виде таблицы отображаются все ссылки. Таблица имеет колонки: короткий код, исходная ссылка, количество переходов и статус. Таблица может быть отсортирована и имеет постраничную навигацию.

На странице Statistics находится строка поиска для ввода ID ссылки или её короткого кода. После поиска выводится подробная информация: количество переходов, переходов по QR, переходов по прямой ссылке, график переходов по дням, QR-код, и исходный URL с разворачиванием.

Администраторская консоль это текстовое меню в терминале. При запуске выводится список из 7 опций. Админист вводит номер опции и проходит по соответствующему сценарию: может увидеть все ссылки, найти конкретную, просмотреть статистику, заблокировать или удалить ссылку.

## Описание backend реализации

Серверная часть реализована на Node.js с использованием фреймворка Express. Основной файл сервера инициализирует приложение, подключается к базе данных, регистрирует маршруты и запускает сервер на порту 3000.

Маршруты разделены на несколько групп. API-маршруты обрабатывают создание, получение и управление ссылками. Маршруты редиректа обрабатывают переход по сокращённой ссылке и запись статистики. Admin-маршруты служат для управления ссылками администратором.

Контроллеры содержат логику обработки каждого маршрута. LinkController обрабатывает создание и получение ссылок. AdminController обрабатывает администраторские операции.

В контроллере LinkController метод createLink принимает исходный URL, проверяет его валидность, генерирует уникальный код, проверяет что это не уже сокращённая ссылка, и сохраняет в базе. Метод redirectLink находит ссылку по коду, записывает переход в таблицу переходов, и редиректит на исходный URL.

Сервисы содержат вспомогательную логику. ShortCodeService генерирует короткие коды используя библиотеку nanoid. QRCodeService преобразует URL в QR-кодовое изображение. SpamProtectionService проверяет что количество запросов с одного IP не превышает лимит.

Модели описывают работу с базой данных. Модель Link имеет методы для создания, поиска, удаления и получения статистики ссылок. Модель Click записывает информацию о переходе. Модель User хранит данные пользователя.

База данных PostgreSQL изначально инициализируется с помощью миграции которая создаёт необходимые таблицы со всеми полями и индексами.

Администраторская консоль реализована как отдельная программа которая если запуститься в терминале предоставляет текстовое меню. Консоль подключается к той же базе данных и показывает информацию о ссылках, позволяет искать, блокировать, удалять и просматривать статистику.

## Описание frontend реализации

Фронтенд реализован на React с использованием инструмента сборки Vite. Приложение состоит из трёх страниц и набора компонентов.

На главной странице находится компонент LinkGenerator который содержит поле ввода. При вводе пользователь получает результат в компоненте LinkResult. Этот компонент показывает исходный URL с функцией разворачивания, сгенерированный код с копированием, и загруженный QR-код.

Страница Dashboard выводит компонент LinkList которые показывает таблицу всех ссылок. Таблица получает данные с сервера через HTTP запрос.

Страница Statistics имеет компонент для поиска и вывода полной информации о выбранной ссылке. Она показывает график переходов по дням, статистику по источникам переходов, и большой QR-код для повторного использования.

Все компоненты используют React hooks для управления состоянием. Функция useState используется для хранения введённых данных, загруженных результатов и состояния разворачивания элементов.

Для связи с сервером используется встроенный API Fetch или библиотека axios. Компоненты отправляют POST запросы при создании ссылки и GET запросы при получении информации.

Стилизация реализована с помощью CSS. Каждая страница и компонент имеет соответствующий CSS файл.

## Описание API

Сервер предоставляет REST API с следующими основными эндпоинтами.

Эндпоинт POST /api/links принимает исходный URL, генерирует и сохраняет короткую ссылку, возвращает ID, сгенерированный код, и полный URL.

Эндпоинт GET /api/links возвращает список всех ссылок с поддержкой пагинации.

Эндпоинт GET /:shortCode обрабатывает переход по сокращённой ссылке, записывает переход в базу, и редиректит на исходный URL.

Эндпоинт GET /api/links/:id/stats возвращает статистику для конкретной ссылки: количество переходов, разбивку по источникам (QR или прямая), и график по дням.

Эндпоинт GET /api/links/:id/qrcode возвращает QR-код в формате Data URL (base64 изображение).

Эндпоинт GET /api/links/:id/qrcode/image возвращает QR-код в виде PNG файла.

Администраторские эндпоинты позволяют просматривать все ссылки, выполнять поиск, блокировать и удалять ссылки.

## Quickstart — кратко и понятно

1. Установить зависимости

```bash
cd backend
npm install

cd ../frontend
npm install
```

2. Запустить backend (API)

```bash
cd backend
npm start
```

3. Запустить frontend (разработка)

```bash
cd frontend
npm run dev
```

Сайт будет доступен по адресу http://localhost:5173

4. Запустить консоль администратора

```bash
cd backend
npm run admin
```

Следуйте меню в терминале

5. Полезные ссылки

- Веб-сайт (локально): http://localhost:5173
- API (локально): http://localhost:3000

6. Примечания

- Файлы проекта находятся в папках `frontend/` и `backend/`.
- В корне репозитория оставлены только метаданные и папки проекта.
- Для деплоя настройте `SERVER_URL` в `.env` и используйте учётные данные для PostgreSQL.
